Modify the create-checkout-session edge function to:

1. **Import Supabase client** at the top of the file (match existing pattern from send-contact-email):
   ```typescript
   import { createClient } from "https://esm.sh/@supabase/supabase-js@2"
   ```

2. **Add price lookup** inside the serve() handler, after parsing the request body:
   - Create a Supabase client using `Deno.env.get('SUPABASE_URL')` and `Deno.env.get('SB_SERVICE_KEY')` (match existing env var names).
   - Filter items to find those with a non-null `id` field: `items.filter((item: any) => item.id)`.
   - Query Supabase: `.from('products').select('id, name, price').in('id', productIds)`.
   - Build a `Map<string, number>` of id->price from the DB results.
   - Replace client-supplied prices: for each item, if `item.id` exists, use `priceMap.get(item.id)` instead of `item.price`. If an item has an `id` but the id is not found in the DB, set its price to `null`.
   - If any items have `null` price (unresolved DB product), return a 400 JSON response: `{ error: "One or more products could not be verified. Please refresh and try again." }` with corsHeaders.
   - Items WITHOUT an `id` (services, legacy cart items) keep their client-supplied price unchanged. This is critical — do NOT require all items to have an id.
   - Only run the Supabase lookup if there are product IDs to look up (`productIds.length > 0`).

3. **Add coupon validation** after price verification:
   - Read `COUPON_CODES` env var: `Deno.env.get('COUPON_CODES') || '{}'`, parse as JSON to get `Record<string, number>` (code -> discount percentage).
   - Read `couponCode` from the request body (the field name changes from `discount` to `couponCode`).
   - If `couponCode` is provided and valid: calculate `discountPercent` from the env var map, then compute `discountAmount = productSubtotal * (discountPercent / 100)` using the SERVER-VERIFIED product subtotal (not client-supplied).
   - If `couponCode` is provided but NOT valid: return 400 JSON: `{ error: "Coupon code \"CODE\" is not valid." }` with corsHeaders.
   - If no `couponCode` provided: `discountAmount = 0`.
   - **Ignore** any `discount` field from the request body entirely — the server calculates the discount from scratch.

4. **Update the rest of the function** to use `verifiedItems` prices and the server-calculated `discountAmount` everywhere the old `items` and `discount` were used. Specifically update:
   - `itemsSubtotal` calculation to use verified items.
   - `discountedSubtotal` to use server-calculated `discountAmount`.
   - `totalAmount` calculation.
   - All Stripe metadata fields (`items_subtotal`, `discount`, `total_amount`) to reflect verified values.

5. **Preserve all existing behavior** for services (the `hasProducts: false` branch). Services don't have UUIDs and use `paymentType`/`paymentAmount` — that path remains untouched.

Important: Keep corsHeaders on ALL responses including error responses (already done in existing code, maintain this).
