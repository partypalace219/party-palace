<objective>
HTML-escape all user-supplied strings in email templates and switch product card rendering from innerHTML interpolation to textContent for user-derived fields.

Purpose: SEC-02 (email XSS) and SEC-05 (product card XSS) are injection vulnerabilities. A malicious customer name like `<script>alert(1)</script>` would execute in the business owner's email client or in other users' browsers viewing product cards with compromised data.

Output: All 3 email-sending edge functions use escapeHtml() on user-supplied strings. All 4 product card rendering locations in app.js use textContent for product.name and product.description.
</objective>

Add the OWASP 5-character HTML escape function to each of the 3 edge functions that send emails, then wrap every user-supplied string interpolated into HTML templates.

**Step 1: Add escapeHtml() to each file** (place near the top, after imports):

```typescript
function escapeHtml(str: string): string {
  return String(str)
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;')
    .replace(/'/g, '&#x27;')
}
```

**Step 2: stripe-webhook/index.ts** — Wrap these fields in all 4 email template functions:

In `sendProductConfirmationEmail()`:
- `${data.customerName}` -> `${escapeHtml(data.customerName)}`
- `${data.orderItems}` -> `${escapeHtml(data.orderItems)}`
- `${shippingInfo}` (contains shippingAddress fields) — escape the individual address fields when building shippingInfo: `escapeHtml(data.shippingAddress.line1)`, `escapeHtml(data.shippingAddress.line2)`, `escapeHtml(data.shippingAddress.city)`, `escapeHtml(data.shippingAddress.state)`, `escapeHtml(data.shippingAddress.postal_code)`

In `sendProductBusinessNotification()`:
- `${data.customerName}`, `${data.customerEmail}`, `${data.customerPhone}`, `${data.orderItems}` — wrap each with escapeHtml()
- Same shippingInfo treatment as above

In `sendBookingConfirmationEmail()`:
- `${data.customerName}`, `${data.eventType}`, `${data.eventDate}`, `${data.venue}`, `${data.orderItems}`, `${data.notes}` — wrap each with escapeHtml()

In `sendBookingBusinessNotification()`:
- `${data.customerName}`, `${data.customerEmail}`, `${data.customerPhone}`, `${data.eventType}`, `${data.eventDate}`, `${data.venue}`, `${data.orderItems}`, `${data.notes}` — wrap each with escapeHtml()

Do NOT escape:
- Numeric values like `data.totalAmount.toFixed(2)`, `data.itemsSubtotal.toFixed(2)` — these are numbers
- `data.depositAmount`, `data.estimatedTotal` — numeric strings from Stripe metadata
- Boolean checks like `data.discount > 0` — these are comparisons, not interpolated strings
- Static text and CSS — these are developer-controlled

**Step 3: send-contact-email/index.ts** — Wrap user fields in both email functions:

In `sendContactEmail()`:
- `${data.name}`, `${data.email}`, `${data.phone}`, `${data.eventType}` — wrap with escapeHtml()
- `${data.selectedProduct!.name}` -> `${escapeHtml(data.selectedProduct!.name)}`
- CRITICAL: `${data.message.replace(/\n/g, '<br>')}` -> `${escapeHtml(data.message).replace(/\n/g, '<br>')}` — escape FIRST, then convert newlines. This prevents HTML injection through the message field while preserving line breaks.

In `sendCustomOrderEmail()`:
- `${data.name}`, `${data.email}`, `${data.phone}`, `${data.product}` — wrap with escapeHtml()
- `${data.description.replace(/\n/g, '<br>')}` -> `${escapeHtml(data.description).replace(/\n/g, '<br>')}` — same pattern: escape first, then newline conversion.
- `${data.orderType}` used in lookups like `orderTypeLabels[data.orderType]` is fine since the lookup returns a developer string, but `${data.orderType}` used directly in the fallback `|| data.orderType` should be escaped: `${orderTypeLabels[data.orderType] || escapeHtml(data.orderType)}`

**Step 4: save-signed-document/index.ts** — Wrap user fields in notification email:

- The `${documentTitle}` is derived from a developer-controlled conditional, not user input — does not need escaping

- Change `<div class="product-name" ...>${product.name}</div>` to `<div class="product-name" ...></div>`
